🎯 Módulo 2: Estructura del proyecto y control de versiones

🧠 Objetivo del módulo

El participante aprenderá a crear un proyecto Angular desde cero con una estructura estándar utilizada en Casa Mecate, y a configurarlo bajo control de versiones utilizando Git, siguiendo buenas prácticas para commits, ramas y colaboración en equipo.

📋 Contenido

1. Inicialización del proyecto

  • Crear un proyecto Angular desde consola: ng new
  • Configurar rutas básicas: app-routing.module.ts
  • Estructura inicial sugerida:
  /src
    /app
      /core
      /shared
      /features
      /pages
      app.module.ts

2. Estructura por carpetas

  • core: servicios globales, guards, interceptores
  • shared: componentes reutilizables, pipes
  • features: módulos funcionales por dominio
  • pages: vistas o páginas principales

3. Control de versiones con Git

  • Inicialización del repositorio: git init
  • Archivo .gitignore para Angular
  • Primer commit
  • Crear rama develop
  • Flujo de ramas: main, develop, feature/nombre

Ejemplos de mensajes de commit:

  feat: agrega estructura inicial del proyecto
  refactor: reorganiza carpetas según arquitectura
  chore: actualiza .gitignore

4. Práctica gamificada

El usuario deberá:

  • Crear la estructura completa del proyecto.
  • Subirlo a GitHub o simularlo en consola.
  • Generar al menos 3 commits válidos con mensajes significativos.
  • Subir evidencia (comandos o capturas).

🧮 Puntaje:

  • 📁 Estructura correcta: 30 pts
  • 🌱 Uso de ramas y commits: 40 pts
  • ✍️ Mensajes bien redactados: 30 pts
